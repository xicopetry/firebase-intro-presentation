Offline

Intermittent Offline

O que acontece quando quando você passa por um túnel?



O "Firebase client" guarda um cache local dos seus dados. Quando o usuario fica offline, o app ainda funciona como o esperado com esse cache local.

Extended Offline

O que acontece quando você anda de avião?



No mobile, Firebase perciste dados na memória local. O app vai continuar a funcionar mesmo quando o usuário reiniciar ele.

Uma linha de código


// Java
Firebase.getDefaultConfig().setPersistenceEnabled(true);

Security

Por default, todos podem ler e escrever 

			
{
"rules": {
	".read": true,
	".write": true,
	}
}

Todos podem ler, mas niguém pode escrever 

			
{
"rules": {
	".read": true,
	".write": false,
	}
}

Segurança em partes específicas do seu Firebase 

			
{
  "rules": {
    ".read": true,
    ".write": false,
    "message": {
      ".write": true 
    }
  }
}

Variáveis especiais para melhorar a autenticação 

						
{
  "rules": {
    ".read": true,
    "message": {
      ".write": "auth !== null",
      ".validate": "!data.exists()"
    }
  }
}

Security

Use $wildcards como rota de parâmetros

						
{
  "rules": {
    ".read": false,
    ".write": false,
    "users": {
      "$userid": {
        ".read": "true",
        ".write": "auth.uid === $userid"
      }
    }
  }
}

AngularFire

A chat app in AngularFire

						
var app = app.module('app', ['firebase']);
app.controller('ChatCtrl', function($scope, $firebaseArray) {
  var ref = new Firebase('https://<your-firebase>.firebaseio.com/messages');
  $scope.messages = $firebaseArray(ref);
});
						
					
						
<html>
  <body ng-app="app">
    <div ng-controller="ChatCtrl">
      <p ng-repeat="message in messages">message.text</p>
    </div>
  </body>
</html>
